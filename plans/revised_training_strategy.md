## 1. Context and Motivation

### 1.1 Project Goal (unchanged)

The project studies **how partial observation affects an RNN's ability to recover ring attractor dynamics**. We train "student" RNNs on data generated by a "teacher" ring attractor simulator, varying:
- The fraction of observed neurons k/N
- The length of the observation window T

We then test whether the student RNN learned a true continuous ring attractor or a degenerate approximation (e.g., discrete fixed points, single point attractor).

### 1.2 What Went Wrong with the Previous Approach

The previous training pipeline (notebooks 11b, 12b) used **teacher-forced next-step prediction**: at every timestep t, the RNN received the ground-truth firing rates x[t] as input and predicted x[t+1].

This approach has a fundamental flaw for learning attractor dynamics: **the ring attractor is a stable system**, meaning x[t+1] ≈ x[t] most of the time. The RNN can achieve low loss by learning a near-identity mapping from input to output, routing information through the input pathway (W_xh) rather than through the recurrent dynamics (W_hh). The recurrent weights — which are exactly where attractor structure would live — are largely unused.

In other words: the RNN solves the training objective without learning any interesting dynamics, because the input acts as a "crutch" that tells the network where the bump is at every step.

### 1.3 The Lorenz63 Contrast

The reference tutorial (Durstewitz group, shPLRNN) works with the Lorenz63 chaotic attractor — a system that never settles down. Every timestep carries rich dynamical information, and the trajectory naturally visits the entire attractor. A single long trajectory of 100,000 steps suffices for training.

The ring attractor is fundamentally different:
- It **converges to stable fixed points** (bumps)
- Multiple fixed points exist (one per angle θ), forming a 1D manifold
- Without external input, the system either maintains a bump or dissolves
- The "interesting" structure is the **geometry of the equilibria**, not the temporal dynamics

This means: (a) we cannot use one long trajectory, (b) we need multiple trials at different angles to cover the manifold, and (c) the training objective must explicitly require the RNN to learn attractor dynamics, not just state maintenance.

## 2. Key Concepts

### 2.1 What Is a Ring Attractor?

A ring attractor is a dynamical system whose stable states form a closed 1D curve (topologically a circle S¹) in state space.

**The state space.** Our network has N = 100 neurons. At any moment, the network state is a vector of firing rates **r** = (r₁, r₂, ..., r_N) ∈ ℝᴺ. This is a point in N-dimensional space.

**The manifold.** Not all points in ℝᴺ are stable states. The ring attractor's stable states are bump-shaped activity patterns parameterized by a single angle θ ∈ [0, 2π). As θ varies, the bump slides around the network, and the rate vector **r**(θ) traces a closed curve through ℝᴺ. This curve is the **attractor manifold** — a 1D object (parameterized by one number, θ) embedded in an N-dimensional space.

"Manifold" means: a smooth surface that locally looks flat. A circle is a 1D manifold (locally looks like a line segment). The surface of a sphere is a 2D manifold (locally looks like a plane). The ring attractor's set of stable states is a 1D manifold that locally looks like a line, but globally wraps into a circle.

### 2.2 Attractor vs. Identity: The Critical Distinction

A ring attractor and the identity matrix both maintain bump states, but they differ fundamentally in how they treat **off-manifold** states:

| Property | Identity (h[t+1] = h[t]) | Ring attractor |
|---|---|---|
| Clean bump at angle θ | Maintained ✓ | Maintained ✓ |
| Noisy/distorted bump | Noise preserved forever | **Noise cleaned up**, bump restored |
| Random initial state | Random state preserved | **Converges to nearest bump** on ring |
| Two simultaneous bumps | Both preserved | **One suppressed** (winner-take-all) |

The difference lies in **stability structure**:

- **Identity matrix**: Neutral stability in ALL N directions. Every state is a fixed point. No attraction toward anything.
- **Ring attractor**: Neutral stability in **1 direction** (along the ring, i.e., changing θ) and **stable dynamics in the other N−1 directions** (perpendicular to the ring). Off-manifold perturbations decay; on-manifold position is preserved.

This is what "attractor" means: states near the manifold are actively **pulled toward** it. The ring is not just a set of fixed points — it's a set of fixed points with a basin of attraction.

### 2.3 What Defines a Ring Attractor Mechanistically?

The teacher simulator uses a cosine connectivity kernel:

$$W_{ij} = \frac{1}{N}\left(J_0 + J_1 \cos(\theta_i - \theta_j)\right)$$

where θᵢ = 2πi/N is neuron i's preferred angle. This is not the only connectivity that produces a ring attractor. The requirements are:

1. **Local excitation**: neurons with similar preferred angles excite each other
2. **Global inhibition**: total activity is bounded, ensuring a single bump
3. **Rotational symmetry**: no angle is privileged

The cosine kernel satisfies all three and has a useful mathematical property: it is **rank-2** (plus a rank-1 uniform component). Using the identity cos(a − b) = cos(a)cos(b) + sin(a)sin(b):

$$W_{ij} = \frac{J_0}{N} + \frac{J_1}{N}\left[\cos(\theta_i)\cos(\theta_j) + \sin(\theta_i)\sin(\theta_j)\right]$$

This decomposes as W = (J₀/N)·**11**ᵀ + (J₁/N)·**c****c**ᵀ + (J₁/N)·**s****s**ᵀ, where **c** and **s** are the vectors of cos(θᵢ) and sin(θᵢ). This rank-2 structure motivates using a low-rank RNN (rank 2) as the student architecture.

### 2.4 Eigenvalue Signature of a Ring Attractor

The Jacobian of the dynamics at any fixed point on the ring has a characteristic eigenvalue spectrum:
- **One eigenvalue at 0** (or at the stability boundary): corresponds to perturbations along the ring (changing θ). This is the neutral direction.
- **N−1 eigenvalues with negative real part**: correspond to perturbations perpendicular to the ring. These decay, providing the attraction.

For the discrete-time RNN, this translates to:
- **One eigenvalue at |λ| = 1**: the ring direction
- **All other eigenvalues |λ| < 1**: the attracting directions

If the trained RNN shows all eigenvalues strictly inside the unit circle, it has learned a stable fixed point (or discrete fixed points), not a ring. If it shows a pair near the unit circle, that's the ring signature.

## 3. Revised Training Strategy

### 3.1 Architecture: Autonomous RNN

The student RNN runs **without external input** during the autonomous phase. Two variants:

**Vanilla RNN:**
$$\mathbf{h}[t] = (1 - \alpha)\,\mathbf{h}[t-1] + \alpha\,\tanh\!\bigl(W_{hh}\,\mathbf{h}[t-1] + \mathbf{b}\bigr)$$
$$\mathbf{y}[t] = W_{hy}\,\mathbf{h}[t]$$

**Low-rank RNN (rank R = 2):**
$$\mathbf{h}[t] = (1 - \alpha)\,\mathbf{h}[t-1] + \alpha\,\tanh\!\bigl(\frac{1}{H}UV^\top\,\mathbf{h}[t-1] + \mathbf{b}\bigr)$$
$$\mathbf{y}[t] = W_{hy}\,\mathbf{h}[t]$$

where U, V ∈ ℝ^{H×R}, and the recurrent matrix W_hh = UV^⊤/H has rank ≤ R.

Parameters: H = hidden_dim (default 100), α = leak rate (hyperparameter), **y**[t] ∈ ℝᴺ is the predicted firing rate vector.

**Input channel (for teacher-forcing phase only):**

During the first K steps of each trial, the RNN receives external input:

$$\mathbf{h}[t] = (1 - \alpha)\,\mathbf{h}[t-1] + \alpha\,\tanh\!\bigl(W_{hh}\,\mathbf{h}[t-1] + W_{xh}\,\mathbf{x}[t] + \mathbf{b}\bigr)$$

where **x**[t] ∈ ℝᵏ is the observed firing rate vector (k ≤ N neurons). After step K, **x**[t] = **0** (input is zeroed, RNN runs autonomously).

### 3.2 Training Data: Three Trial Groups

All data is generated from the ring attractor simulator with parameters:
- N = 100 neurons, J₀ = −2.0, J₁ = 4.0
- τ = 100, dt = 0.01, σ = 0.1 (noise)
- ReLU nonlinearity
- Spikes generated via Poisson process, binned and smoothed

**Important**: all training data should be **standardized** to zero mean and unit variance per neuron (computed across all trials and timesteps in the training set). Store the mean μᵢ and std σᵢ per neuron for de-standardization during evaluation.

#### Group A: Bump Maintenance (~36 trials)

**Purpose:** Teach the RNN the manifold — what bump patterns exist at each angle.

**Generation procedure:**
1. For each trial, select θ_target uniformly from {0°, 10°, 20°, ..., 350°}
2. Run simulator with a cue at θ_target for T_cue = 2000 integration steps
3. Remove cue, continue for T_settle = 500 integration steps (confirm stability)
4. Record the next T_record integration steps as the training trajectory
5. Bin and smooth as before (bin_factor = 50, smoothing_window = 3)

**Training data per trial:**
- `rates_init`: firing rate vector at start of recording (shape: N)
- `rates_trajectory`: the subsequent T_bin timesteps (shape: T_bin × N)

**During training:** Teacher-force steps 1 through K, then autonomous for steps K+1 through T_bin. Loss computed on steps K+1 through T_bin only.

#### Group B: Noisy Bump Convergence (~36 trials)

**Purpose:** Teach transverse stability — the RNN must clean up off-manifold perturbations.

**Generation procedure:**
1. For each trial, select θ_target and generate a converged bump (same as Group A steps 1-3)
2. At the start of recording, **add noise** to the firing rates: r_noisy = r_converged + ε, where ε ~ 𝒩(0, σ_perturb²·I). Choose σ_perturb so the perturbation is visible but doesn't destroy the bump (e.g., σ_perturb = 0.2 × max(r_converged))
3. Continue the simulator from the perturbed state for T_record steps
4. Record the trajectory (which shows convergence back to a clean bump)

**During training:** Same as Group A — teacher-force first K steps, autonomous after, loss on autonomous phase.

#### Group C: Random Initialization Convergence (~36 trials)

**Purpose:** Teach the basin of attraction — the RNN must converge to the manifold from arbitrary states.

**Generation procedure:**
1. Initialize the simulator from a **random state** (random firing rates, not from a cue)
2. Run for T_record steps, during which the simulator converges to some bump at angle θ_converged
3. Record the full trajectory including the transient convergence phase

**During training:** Teacher-force the first K_long steps (may need K_long > K for Group A/B, since the initial state is further from the manifold). Loss on autonomous phase.

**Note:** This group has a state-space mismatch concern (see Section 6, Critique 1). Consider implementing Group C only after Groups A and B produce a working ring. Group C is important for attractor dynamics but not strictly necessary for the first milestone.

### 3.3 Training Procedure

#### Initialization
- RNN weights initialized randomly (Xavier/Glorot)
- h[0] = **0** for all trials (the teacher-forcing phase will set up the appropriate hidden state)

#### Teacher-Forcing Phase (steps 1 to K)
- The RNN receives observed firing rates **x**[t] as input
- K is drawn randomly per trial: K ~ Uniform(K_min, K_max), e.g., K ∈ [5, 20]
  - Randomizing K prevents the RNN from learning a fixed transition artifact
- No loss is computed during this phase

#### Autonomous Phase (steps K+1 to T)
- Input is set to **x**[t] = **0**
- The RNN evolves purely through its recurrent dynamics
- Loss is computed during this phase

#### Loss Function

$$\mathcal{L} = \frac{1}{|\mathcal{B}|} \sum_{i \in \mathcal{B}} \sum_{t=K_i+1}^{T} w(t - K_i) \cdot \|\mathbf{y}_i[t] - \mathbf{r}_i^{\text{true}}[t]\|^2$$

where:
- 𝓑 is the batch of trials
- Kᵢ is the (randomly chosen) teacher-forcing length for trial i
- w(t − Kᵢ) is an optional weighting function

**Weighting options** (choose one, or compare):
- **Uniform**: w(s) = 1 for all s. Simplest.
- **Convergence-weighted**: w(s) = w_conv if s ≤ T_conv, else 1. E.g., w_conv = 5 for the first T_conv = 30 autonomous steps. This up-weights the convergence phase so the attractor dynamics aren't drowned out by the maintenance phase.

#### Optimizer and Hyperparameters

| Parameter | Default | Notes |
|---|---|---|
| Optimizer | Adam | |
| Learning rate | 1e-3 | May need tuning |
| α (leak rate) | 0.5 | Controls RNN timescale; should roughly match τ of simulator |
| Hidden dim H | 100 | Matches N for simplicity |
| Rank R (low-rank only) | 2 | Matches cosine kernel rank |
| Epochs | 1000–2000 | Monitor convergence |
| Gradient clipping | 1.0 | Max norm, to handle long autonomous rollouts |
| K_min, K_max | 5, 20 | Teacher-forcing window (randomized per trial) |
| T_bin (trial length) | 100–200 | After binning; long enough to test maintenance |

### 3.4 Partial Observation

After the full-observation (k = N) pipeline works, introduce neuron dropout:
- Randomly select k neurons to observe (k/N ∈ {1.0, 0.5, 0.25, 0.1})
- During teacher-forcing, the RNN receives only the k observed neurons: **x**[t] ∈ ℝᵏ
- The RNN still predicts all N neurons: **y**[t] ∈ ℝᴺ
- W_xh has shape (H, k) — changes with observation level
- Everything else stays the same

## 4. Evaluation and Metrics

### 4.1 Predictive Metrics (necessary but not sufficient)

**Train/Test MSE:** Standard loss on held-out trials. Should decrease with training. But low MSE does NOT guarantee correct dynamics (the identity solution also has low MSE on Group A).

**Circular angle error:** Decode angle from predicted rates using population vector:

$$\hat{\theta}[t] = \text{atan2}\!\left(\sum_i y_i[t]\sin(\theta_i),\;\sum_i y_i[t]\cos(\theta_i)\right)$$

Compare to ground-truth θ via circular distance: $d(\hat{\theta}, \theta) = \min(|\hat{\theta} - \theta|, 2\pi - |\hat{\theta} - \theta|)$.

### 4.2 Autonomous Fixed-Point Analysis (the core test)

This is the primary evaluation. It tests whether the trained RNN has a ring attractor in its recurrent dynamics.

**Procedure:**
1. Initialize M = 300 random hidden states: **h**₀ ~ 𝒩(**0**, σ_init²·I)
2. Run each autonomously (zero input) for T_auto = 500 steps
3. Collect the final hidden states **h**_final and decode their angles θ_final

**Ring score metrics:**

- **Spread**: mean ‖**h**_final‖₂ across all M initializations. Should be significantly > 0 (non-trivial fixed points, not origin collapse).

- **Uniformity**: 1 − |mean(exp(iθ_final))|. Equals 1 if converged angles are uniformly distributed around the ring; equals 0 if all states converge to the same angle.

- **Circularity**: Project converged hidden states onto PC1, PC2. Compute ratio of the smaller to larger standard deviation: min(std_PC1, std_PC2) / max(std_PC1, std_PC2). Equals 1 for a circle; near 0 for a line or point.

**Visual diagnostic:** PCA scatter plot of converged hidden states, colored by decoded θ. A successful ring shows a smooth circle with a rainbow color gradient.

### 4.3 Generalization Test (ring vs. discrete points)

**Purpose:** Distinguish a true continuous ring from a polygon of discrete fixed points.

**Procedure:**
1. Train on S angles (e.g., {0°, 10°, 20°, ..., 350°})
2. Test on S intermediate angles (e.g., {5°, 15°, 25°, ..., 355°})
3. For each intermediate angle θ_test:
   a. Initialize the RNN at a bump corresponding to θ_test (via teacher-forcing)
   b. Run autonomously for T_gen = 500 steps
   c. Record θ_final

**Metric:** Angular drift = θ_final − θ_test for each test angle.

- **True continuous ring:** drift ≈ 0 (or small random walk)
- **Discrete fixed points:** drift systematically toward the nearest training angle. Plot drift vs. initial angle — should be flat for a ring, sawtooth for discrete attractors.

### 4.4 Eigenvalue Analysis

Compute eigenvalues of the effective recurrent Jacobian at fixed points on the ring.

For a vanilla RNN at fixed point **h***:
$$J = (1 - \alpha)I + \alpha\,\text{diag}(1 - \tanh^2(W_{hh}\mathbf{h}^* + \mathbf{b}))\,W_{hh}$$

For the low-rank RNN, substitute W_hh = UV^⊤/H.

**Expected spectrum for a ring attractor:**
- One eigenvalue pair near |λ| = 1 (the ring directions, cos/sin modes)
- All others |λ| < 1 (attracting, transverse directions)

**Failure signatures:**
- All |λ| < 1: single stable fixed point, no ring
- Multiple eigenvalues at |λ| = 1: higher-dimensional attractor or instability
- Eigenvalues |λ| > 1: unstable dynamics

### 4.5 Singular Value Analysis (low-rank RNN only)

For the low-rank RNN, compute the singular values of W_hh = UV^⊤/H.

**Expected:** Two dominant singular values (reflecting the rank-2 cosine structure), sharp drop after index 2.

Optionally: reorder the rows/columns of W_hh by each hidden neuron's "preferred angle" (found from the readout weights or tuning curves). If the reordered matrix shows cosine-kernel-like banded structure, the RNN has learned the correct connectivity motif.

## 5. Milestones

### Milestone 0: Check Existing Results
Before implementing anything new, inspect the existing 12b notebook results (if any). Run the autonomous fixed-point analysis on the already-trained models. If there's partial ring structure, the previous approach may have been closer than we think.

**Exit criterion:** Clear understanding of what the current models learned (ring, discrete points, single point, or nothing).

### Milestone 1: Full-Observation Ring Recovery
Train the autonomous RNN (both vanilla and low-rank) on Group A + Group B trials with k/N = 1.0. Achieve a clear ring in the fixed-point analysis.

**Exit criterion:** Uniformity > 0.8, circularity > 0.7, and the generalization test shows drift < 5° for intermediate angles. Do NOT over-optimize — once this works, move on.

### Milestone 2: Partial Observation Experiments
Sweep k/N ∈ {1.0, 0.5, 0.25, 0.1}. For each, train the RNN and compute all metrics from Section 4.

**Exit criterion:** Clear characterization of how ring score degrades with k/N. Identify the threshold where the ring breaks down.

### Milestone 3: Group C and Robustness (if time allows)
Add Group C (random initialization) trials. Add noise injection during training. Characterize whether these improve the attractor basin.

### Milestone 4: Real Data (Phase 5)
Apply the pipeline to DANDI:000939 head-direction cell recordings. This reuses the same student RNN and evaluation code.

## 6. Known Limitations and Risk Mitigations

### Critique 1: Group C State-Space Mismatch
The simulator's "random state" lives in rate space; the RNN's hidden state lives in a different learned space. Setting h[0] = random rates may not be meaningful for the RNN.

**Mitigation:** Use teacher-forcing (not direct h[0] encoding) for Group C. Defer Group C to Milestone 3 — it's not needed for the first ring recovery.

### Critique 2: Loss Dominated by Maintenance Phase
If trials are 200 steps and convergence takes 20 steps, 90% of the loss gradient comes from the maintenance phase, which rewards identity-like behavior.

**Mitigation:** Use convergence-weighted loss (Section 3.3). Also consider shorter trials. Monitor loss contributions from early vs. late timesteps.

### Critique 3: Scope Creep
The real project question is about partial observation, not "how to train an RNN to learn a ring." The training pipeline is a prerequisite, not the goal.

**Mitigation:** Strict milestone progression. Do not over-optimize the full-observation case (Milestone 1). Once the ring appears, move to partial observation (Milestone 2).

### Critique 4: Maybe the Original Approach Wasn't Fundamentally Wrong
The original next-step prediction approach might have partially worked. We may be rebuilding from scratch unnecessarily.

**Mitigation:** Milestone 0 — check existing results first. If partial ring structure exists, consider whether hyperparameter tuning (more epochs, different α, different learning rate) could fix it without a complete redesign.

### Critique 5: Teacher-Forcing Cutoff Artifact
The sharp transition from teacher-forced to autonomous at step K may create artifacts.

**Mitigation:** Randomize K across trials (K ~ Uniform(K_min, K_max)). Optionally taper the input magnitude smoothly over a few steps at the transition.

### Critique 6: Ring vs. Polygon Ambiguity
Training on 36 discrete angles could produce 36 discrete fixed points arranged in a circle, visually indistinguishable from a ring in PCA.

**Mitigation:** The generalization test (Section 4.3) explicitly distinguishes these. Define pass/fail criteria before running experiments, not after.

### Critique 7: Data Normalization
Firing rates from the simulator may have uneven scales. Unnormalized data biases the MSE toward high-firing neurons.

**Mitigation:** Standardize per neuron to zero mean, unit variance. Store normalization parameters.

### Critique 8: No Noise in RNN During Training
The simulator has noise; the RNN is deterministic. This model mismatch may affect robustness of learned dynamics.

**Mitigation:** Consider adding noise injection to the RNN update rule: h[t] += σ_rnn · ε[t] where ε ~ 𝒩(0, I). Defer to Milestone 3 — useful for robustness but not needed for initial ring recovery.

### Critique 9: Gradient Issues with Long Autonomous Rollouts
Backpropagating through T autonomous steps can cause vanishing/exploding gradients, since the gradient flows through T applications of W_hh with no external input injection.

**Mitigation:** Gradient clipping (max norm = 1.0). Keep the autonomous phase moderate length (100–200 steps). Monitor gradient norms during training. If needed, use truncated BPTT (backprop through only the last T_trunc steps).

## 7. Summary of Changes from Previous Pipeline

| Aspect | Previous (11b/12b) | Revised |
|---|---|---|
| RNN receives input at every step | Yes (ground-truth rates) | Only first K steps (teacher-forcing) |
| Autonomous phase | Only during evaluation | During training (core of the loss) |
| Training data | Mixed trial types (cue, noise, perturbation) | Three targeted groups (A: maintain, B: denoise, C: converge) |
| Trial concatenation | Independent trials in batch | Same — independent trials, h reset between |
| Loss function | MSE on all timesteps | MSE on autonomous phase only, optionally weighted |
| Teacher-forcing length | All timesteps | K ~ Uniform(5, 20), randomized per trial |
| Data normalization | Not standardized | Standardize per neuron (zero mean, unit variance) |
| Primary evaluation | Test MSE + angle error | Autonomous fixed-point analysis + generalization test |
| Cue as input | Not used in RNN | Not used (autonomous architecture) |

## Appendix B: Quick Reference — Equations

**Simulator dynamics (continuous-time):**
$$\tau\,\frac{d\mathbf{r}}{dt} = -\mathbf{r} + \phi(W\mathbf{r} + \mathbf{I}_{\text{ext}} + \sigma\,\boldsymbol{\xi}(t))$$

**Simulator weight matrix:**
$$W_{ij} = \frac{1}{N}\left(J_0 + J_1\cos(\theta_i - \theta_j)\right), \quad \theta_i = \frac{2\pi i}{N}$$

**RNN update (autonomous phase):**
$$\mathbf{h}[t] = (1-\alpha)\,\mathbf{h}[t-1] + \alpha\,\tanh(W_{hh}\,\mathbf{h}[t-1] + \mathbf{b})$$

**RNN update (teacher-forcing phase):**
$$\mathbf{h}[t] = (1-\alpha)\,\mathbf{h}[t-1] + \alpha\,\tanh(W_{hh}\,\mathbf{h}[t-1] + W_{xh}\,\mathbf{x}[t] + \mathbf{b})$$

**RNN readout:**
$$\mathbf{y}[t] = W_{hy}\,\mathbf{h}[t]$$

**Angle decoding (population vector):**
$$\hat{\theta} = \text{atan2}\!\left(\sum_{i=1}^{N} y_i\sin\theta_i,\;\sum_{i=1}^{N} y_i\cos\theta_i\right)$$

**Loss (per batch):**
$$\mathcal{L} = \frac{1}{|\mathcal{B}|}\sum_{i \in \mathcal{B}}\sum_{t=K_i+1}^{T} w(t-K_i)\;\|\mathbf{y}_i[t] - \mathbf{r}_i^{\text{true}}[t]\|^2$$
